# 실행

uv add gradio

uv run app.py

# 역할
1. Engineering Lead (설계 팀장)

    역할: 엔지니어링 팀을 지휘하고 백엔드 개발을 위한 상세 설계를 준비합니다.

    목표: 상위 수준의 요구사항을 분석하여 클래스 구조, 메서드 시그니처, 기능을 정의한 상세 설계서를 작성합니다.

    산출물: {module_name}_design.md (마크다운 형식의 설계 문서)

    특징: seasoned 설계자로서 명확하고 간결한 설계를 지향합니다. (GPT-4o 활용)


2. Backend Engineer (백엔드 개발자)

    역할: 설계 팀장의 디자인을 코드로 구현하는 파이썬 전문가입니다.

    목표: 설계 지침을 엄격히 따라 요구사항을 충족하는 단일 파이썬 모듈을 작성합니다.

    산출물: {module_name} (실행 가능한 순수 파이썬 코드)

    특징: 효율적이고 깨끗한 코드를 작성하며, 바로 테스트가 가능한 자립형 모듈을 만듭니다. (Claude 3.7 활용)


3. Frontend Engineer (프론트엔드 개발자)

    역할: 백엔드 기능을 시연하기 위한 Gradio UI 전문가입니다.

    목표: 백엔드 클래스를 임포트하여 기능을 테스트해 볼 수 있는 매우 단순한 프로토타입 UI를 만듭니다.

    산출물: app.py (Gradio 기반의 UI 코드)

    특징: 같은 디렉토리 내의 백엔드 모듈을 활용하여 1인 사용자를 위한 데모 화면을 구성합니다.


4. Test Engineer (테스트 엔지니어)

    역할: 코드 품질을 검증하는 QA 및 소프트웨어 개발자입니다.

    목표: 백엔드 모듈의 정상 작동을 확인하기 위한 단위 테스트(Unit Test)를 작성합니다.

    산출물: test_{module_name} (단위 테스트 파이썬 코드)

    특징: 견고한 테스트 케이스를 통해 코드의 안정성을 확보합니다.

    

    [작업 태스크 및 흐름]
    Design Task: 요구사항을 바탕으로 모듈 내 클래스와 함수를 정의 (Markdown 출력)

    Code Task: 디자인을 실제 코드로 구현 (Raw Python Code 출력)

    Frontend Task: 구현된 백엔드를 Gradio로 연결하여 시연 (Raw Python Code 출력)

    Test Task: 구현된 백엔드에 대한 유닛 테스트 작성 (Raw Python Code 출력)
